# Switchyard Public Donor Boundary Ledger

This public ledger keeps only the outward-facing donor boundary summary.

It is intentionally thinner than any maintainer-only review packet and exists
to answer one question:

> Which donor posture is allowed for each public surface family?

## Public Summary

| Surface family | Allowed posture | Must stay out |
| --- | --- | --- |
| Runtime/operator shells | dense-but-calm, evidence-first, local-first | docs shell takeover, launcher-as-identity, hosted SaaS theater |
| Utility chrome | compact assistive controls only | becoming the whole page identity |
| Public docs/help surfaces | answer-first editorial routing, proof restraint | runtime cockpit styling, file-cabinet IA, decorative proof inflation |

## Use This Ledger For

- checking whether a proposed public-facing change stays inside the approved
  surface family
- reminding contributors that donor posture is not donor product identity
- keeping the public repo from advertising deeper maintainer doctrine than it
  needs

## Do Not Use This Ledger For

- row-by-row pixel approval
- reopening donor selection
- inferring that any donor product worldview is part of Switchyard

If a change needs finer review than this public ledger can provide, that review
belongs in a maintainer-only shelf outside the public repo plane.
