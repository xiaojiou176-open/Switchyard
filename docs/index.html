Choose the shortest truthful route before you open the fuller atlas.

Switchyard is not a chat product and not a hosted magic shell. This docs front door works best when it behaves like a reviewer map: first understand the runtime identity, then the first-success path, then the proof, distribution, or API lane you actually need.

Current rule of thumb: repo-side green and proof-pack green still do not erase credentialed workstation reality or later listing work.

Read it in this order
  1. 30-second overview when you need the product sentence fast.
  2. First success when you want one bounded route that actually runs.
  3. Public proof pack when the question becomes “what is really proved”.
  4. Service HTTP reference when the question becomes “what can I call”.

This page is intentionally still docs-first. It is here to route a reviewer, not to invent a fake app UI.
